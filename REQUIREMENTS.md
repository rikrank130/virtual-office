# シニア交流バーチャルスペース 要件メモ（v0 / プロトタイプ）

## コンセプト
oVice / Gather のような**2Dバーチャルオフィス**を、シニア（高齢者）が無理なく使える形にしたWeb交流スペース。
広場の中を自分のアバターで歩き、**近くにいる人と自然に会話が始まる**「距離ベースの交流」を中心にする。

## ターゲット / 前提
- 利用者: 60〜80代を想定。スマホ・タブレット・PCいずれも。
- ITに不慣れな人が多い前提で、**操作を徹底的に単純化**する（今回の最重視ポイント）。
- 説明書なしで「触れば分かる」ことを目標にする。

## コア体験（MVPで検証したいこと）
1. **広場に入る**: 名前を入れる/選ぶだけで即入室（アカウント登録なし）。
2. **歩く**: 行きたい場所を**タップ/クリックするだけ**でアバターが移動（キー操作不要）。
3. **近づくと会話**: 他の人に近づくと自動で会話パネルが開く。離れると閉じる。
4. **話す**: 大きな文字でチャット。定型あいさつボタンも用意。
5. **居場所（ゾーン）**: 「お茶の間」「囲碁・将棋」「健康体操」「写真じまん」など話題ごとの場所。

## シニア向けUI原則（操作をとにかく単純に）
- 1画面1目的。ボタンは最小限・大きく・離して配置。
- 文字は特大、配色は高コントラスト。
- 移動は「行きたい所をタッチ」だけ。複雑なジョイスティック/キー操作はなし。
- 専門用語を使わない（「ルーム」→「お部屋」「広場」など）。
- 失敗しても怖くない。元に戻る/閉じるが常に分かる。

## プロトタイプの範囲（今回作るもの）
- React + Vite の単一ページ。サーバ・通信なし、**ダミーデータ**で動作。
- 2Dの広場、自分のアバター、数名のダミーのシニアアバター（一部はゆっくり動く）。
- タップ移動、距離ベースの会話パネル、定型あいさつ、簡単な自動返信。
- 話題ゾーンのラベル表示。

## 今回やらないこと（次フェーズ）
- 音声・ビデオ通話。
- アカウント・認証、データ保存、見守り/通報、音声読み上げ。
- スマホ最適化の作り込み（プロトは動作確認レベル）。

## 確認できたら次に決めること
- 音声/ビデオ通話は必要か（oVice的な「近づくと声が聞こえる」を入れるか）。
- 見守り・安否確認など「安心」機能の優先度。
- 運営（モデレーター/ボランティア）の関わり方。
